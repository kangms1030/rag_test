## 1. 왜 바꿨나 (동기)

test_1 실측에서 드러난 두 가지 치명적 한계:

1. **VLM 표 숫자 오독**: VLM이 조밀한 표 이미지를 직접 판독하다 숫자를 자주 틀림. 8회 수치 관측 중
   **4회 오류**(예: Cat.5 `10Mbps`→`100Mbps` 오독, 전화번호 `1899-0979`→`1899-0970`, 원문에 없는
   `6.933Gbps` 환각). bbox(위치)는 맞아도 셀 값 전사에서 틀리는 구조적 약점(`test_1 REPORT.md §5 A1`).
2. **극심한 지연**: 질문마다 최대 24페이지 VLM 요약 + 4페이지 시각청킹 = **~29 VLM 호출/문항**,
   냉시작 시 문항당 **13~15분**.

→ "표는 **결정론적 도구로 텍스트화**한 뒤, 모델에겐 **전사만** 시키자"는 방향으로 재설계.

## 2. 핵심 변경 (Before → After)

| 항목 | test_1 | test_2 | 이유 |
|---|---|---|---|

| 파서 | PyMuPDF+pdfplumber (렌더/텍스트) | **+ MinerU 3.4.4 pipeline**(레이아웃/표/OCR) | 표를 **마크다운으로 결정론 추출** |
| 표 처리 | VLM이 이미지로 판독 | **MinerU 마크다운을 LLM이 전사** | VLM 오독 제거(수치 8/8 정확) |
| 질문당 모델호출 | ~29 VLM(+1~2 LLM+11~33 embed) | **2회**(embed 1 + answer 1) | ~40배 속도 |
| 쿼리분석(LLM) | 있음 | **제거** | 결정론 라우팅으로 대체 |
| VLM 요약·시각청킹 | 있음 | **제거** | 사전 텍스트 색인이 대체 |

## 3. 무엇이 좋아졌나 (효과)

- **수치 정확도**: keyword_recall 0.646 → **0.736**, 수치 오류 4회 → **0회**.
- **속도**: 문항당 868s → **20~37s**(**~40배**). 질문당 호출 29 VLM → **2회**.
- **검색 정확도**: doc/page match 여전히 100%(3문항). no_catalog가 오히려 doc_hit 우세(카탈로그 효과 반전).
