## 실험판 정의
- **A. 기존판(대조군)** — rag3, 로컬 gemma4:12b. varfix baseline. 무수정 보존.
- **B. 로컬 개선판** — rag3x + `verify_skip`(조건부 groundedness 생략) + `adaptive_trim`(저점 페이지 컷)

- **C. Flash-lite판** — rag3x + `gemini-3.1-flash-lite`(생성·검증), **검색·임베딩·리랭커는 로컬 불변**.
  20 RPM 스로틀(3.2s), 장애 시 로컬 12b 폴백. (계획의 gemini-2.5-flash는 API 키에 404→lite 대체)
- **D. Flash+분해판** — C + 종합형 라우팅/분해검색/문장인용(`rag3x/decompose.py`). 종합형 평가셋에서 측정 완료(§종합형).

### 주석
1. **C의 답변율은 내용기반(kw>0) 0.903.** machinery answered_rate는 0.258로 나오나 이는 오라벨:
   Flash의 hedged 답변("확인할 수 없으나 …있습니다")이 12b용 `is_abstain()`를 오탐→매 문항
   spurious rollback + conf=abstain. 실제로는 31문항 중 28개가 grounded 답변(unsup=0). (PHASE6_P2 §중대발견)
2. 검색·임베딩·리랭커가 A/B/C 모두 **동일 로컬 스택**이라 page_hit/doc_hit 불변(설계상 보장).
3. 무관거절은 로컬 검색 floor에서 결정(모델 무관)—C도 5/5 거절, API 호출 0.
4. 개방=baseline abstain 문항이 grounded 답변으로 전환. C: core_002/008·vp_006/007(kw 0→0.6~1.0),
   vp_013(카드UI)만 잔존. B: core_002/008만. (환각 아님, unsup=0)
5. **B의 max 181s(vp_007, deadline 도달).** retry 예산 축소(2→1)로도 꼬리 못 잡음—로컬 12b의
   구조적 한계(호출당 10~15s×다중호출). 로컬 개선으로 "정상 30s/최악 90s" 목표 **미달**.
6. C는 Gemini 호출수. spurious rollback 탓에 이상적 2회보다 많음(오탐 처방 시 감소 여지).
7. lite tier **가정 요율**(in $0.10/out $0.40 per 1M) × 실측 토큰. 무료 티어면 사실상 ₩0(20 RPM 제약).

## 관찰·추천 (참고용 — 결정 아님)

- **속도·정확도·비용 모든 축에서 C(Flash-lite)가 A/B를 지배.** 특히 "전 문항 20초"는 로컬로는
  구조적으로 불가하고 C에서만 달성. 파싱천장(유일 미달 지표)도 C가 4/5 개방.
- **B(로컬개선)의 이득은 미미**(kw +0.065, avg −7.6s)하고 꼬리는 오히려 악화. 로컬 유지를 택할
  경우의 상한을 보여주는 참고치.
- **C의 유일 약점 2가지**: (i) 외부 API 의존(20 RPM·네트워크) — 단 폴백=로컬 12b로 가용성 확보,
  (ii) verify 계층 오라벨(회피 오탐). **(ii)는 D의 문장인용으로 처방됨**(답변율 0.583→0.917, 환각 0 유지).
- **D(Flash+분해)는 진짜 문서간 질문에서만 추가 이득**(문서간 0.625→0.917), 단일형엔 불필요.
  라우팅을 "문서간"에 정밀 진입시키고 종합형 문항을 늘리면 D의 평균 우위가 뚜렷해질 것.

## 종합 추천 (참고용 — 결정 아님)

- **가장 강한 후보: C(Flash-lite)를 기본 생성·검증으로, 문서간 질문에 한해 D(분해+문장인용) 라우팅.**
  속도(전 문항<20s)·정확도(kw+0.19)·비용(~$0.0004/문항)·파싱천장(4/5 개방)에서 A/B를 지배하고,
  D의 문장인용이 Flash의 유일 약점(오라벨)까지 해소. 절대지표(환각 0·무관거절 100%)는 전 실험판 유지.
- **로컬 유지(B)를 택할 경우 상한**: kw 0.593·avg 55.8s로 A 대비 이득 미미, 꼬리 미해결. "전 문항 20s"·
  파싱천장은 로컬로 구조적 불가.
- **운영 관점 '병행'**: C/D는 전부 설정 플래그로 on/off이며 OFF면 A와 동일 동작(등가성 증명). 외부 API
  장애 시 로컬 12b 자동 폴백. 즉 "Flash 기본 + 장애 시 로컬"의 병행 운영이 그대로 가능.
