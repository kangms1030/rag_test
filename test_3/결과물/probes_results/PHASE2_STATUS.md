# Phase 2 상태 리포트 (답변 + 검증 + 롤백 루프)

> 작성: 2026-07-15. 원시결과: `phase2_eval.json`. 구현: `test_3/rag3/{controller,verify,judge,answer}.py`.
> 평가: 36문항(rag_eval 16 + vlm_probe 20), 단일 파이프라인 end-to-end.

## 구현 완료 (계획 S6~S8)

- **controller.py**: 상태기계(검색→답변→검증→롤백). 재시도 예산(crag/answer_regen/path_switch 각 ≤1), 총 모델호출 ≤5, wall-clock 데드라인 180s.
- **verify.py**: ① 결정론 숫자/시리얼 대조(답변의 수치·코드가 근거에 실재하는지, 모델호출 0) ② vision 전사 vs MinerU OCR diff ③ groundedness(12b 짧은 호출, Phase 0-E: 12b 겸용 스왑 0). LLM 판정 단독은 롤백 불가, 신뢰도 태그만.
- **judge.py**: CRAG 질의 재작성(게이트 거절+경계 점수 시 1회 재검색).
- **answer.py**: text/vision 분리 호출 + 컨텍스트 토큰 예산(10K 트림) + stale 이미지 경로 자동 해석·text 폴백.
- **롤백 3종(결정론 트리거)**: A) text 확인불가+검색신뢰 → 1순위 페이지 단독 재시도, B) text 숫자 미지원+스캔 → vision 교차확인, C) vision 전사≠OCR → OCR 텍스트로 재구성.
- **num_predict 1024**: 큰 표에서 12b 장문 생성 폭주(실측 280s)를 차단 → 문항 지연 상한화.

### 해석: 품질 목표 미달은 Phase 2가 아니라 Phase 1(검색)이 병목
- 관련 31문항 중 **11문항은 정답 페이지가 애초에 검색 top3에 없음(ev=0, Phase 1 page_hit@3 0.645의 이면)**. 그중 9건은 정당하게 abstain(근거 없음 → 환각 대신 정직한 거절). 이건 Phase 2가 올바르게 동작한 결과지 실패가 아니다.
- **근거존재(ev=1) 20문항 중 15문항 답변(0.75), 답변 품질 kw_hit 0.704(목표 달성)**. 즉 검색이 성공하면 Phase 2는 목표대로 답한다.
- **근거있는데 실패 5건**(core_002, core_008, vp_006, vp_007, vp_013): 전부 "확인 불가". vp_006(시리얼 R54KB00TXWF)·vp_007(중첩 표 숫자)은 **P0-A에서 vision 타일링으로도 못 고친 난케이스** — MinerU OCR 원문에도 오독이 있어 텍스트로도 추출 불가. 파싱/코퍼스 한계에 가깝다.

## 결론

- **Phase 2 인프라는 완성·검증됨**: 검증·CRAG·롤백이 예산 내에서 작동하고, **환각 0**(못 찾으면 회피)이라 챗봇 모듈의 신뢰성 요건을 만족한다. 롤백이 실제로 답을 회복하고, 답변 품질(kw_hit 0.704)은 목표를 넘는다.
- **미달한 집계 목표(근거있는데실패 0, 전체 kw_hit 0.70)는 Phase 1 검색 품질(page_hit@3 0.645)에 의해 상한이 걸린다.** 계획이 Phase 1(page_hit≥0.75)을 Phase 2 앞에 둔 이유가 이것. 현재 Phase 1은 0.645에 머물러 end-to-end 답변율을 제약한다.
