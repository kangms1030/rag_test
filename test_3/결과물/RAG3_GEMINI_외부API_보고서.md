# RAG3 외부 API(Gemini) 활용 RAG — 변경점 · 성능 비교 보고서

> 작성: 2026-07-19. 대상: 완성형 로컬 RAG **RAG3(test_3)** 를 기반으로, LLM 호출 자리만 외부 API
> (Gemini Flash-lite)로 치환한 **대조군 실험판**. 목적: "외부 API RAG가 로컬 모델 대비 얼마나 빠른가"를
> 동일 조건에서 실측 제시. 상세 근거: `probes/results/PHASE6_P2.md · PHASE6_P3.md · PHASE6_COMPARISON.md`.
>
> **원칙**: 기존 rag3 코드는 한 줄도 수정하지 않았다. 모든 구현은 별도 패키지 `rag3x/`(병행 포크)에 있고,
> 실험 플래그가 전부 OFF면 rag3와 **동작이 완전히 동일**함을 실측 증명했다.

## 2. 기존 RAG3에서 무엇이 달라졌나

### 2.1 바뀐 것 — LLM 호출 자리만 Gemini로
RAG3 파이프라인(S1~S8)에서 **로컬 gemma4:12b가 담당하던 LLM 호출 자리만** Gemini Flash-lite로 치환:
- **S6 답변 생성** (text/vision)
- **S7 검증** (groundedness 판정)
- **CRAG 질의 재작성**

`rag3.models.Backend` 인터페이스(`chat_text`/`chat_vision_text`/`embed`)를 그대로 구현한 `GeminiBackend`를
만들어, **컨트롤러 로직을 전혀 수정하지 않고** 백엔드만 갈아끼웠다.

### 2.2 바뀌지 않은 것 — 검색 스택은 100% 로컬 유지
- **임베딩(embeddinggemma) · 하이브리드 검색(BM25+dense RRF) · 리랭커(bge-reranker-v2-m3) ·
  small-to-big 페이지 승격 · 컨트롤러 상태기계**는 전부 기존 로컬 그대로.
- 이유: 검색 품질 지표(page_hit@3 **0.774**, doc_hit@3 **1.0**)는 로컬 스택의 성과다. 임베딩을 API로
  바꾸면 2,562개 벡터 전체 재색인 + API 영구 종속이 생기므로 **의도적으로 로컬 유지**.
- 결과: **검색 지표(page_hit/doc_hit)는 로컬판과 완전히 동일**. 달라진 건 "찾은 근거로 답을 쓰는 속도·품질"뿐.

### 2.3 외부 API 운영을 위해 새로 넣은 장치 (`rag3x/gemini_backend.py`)
- **키 관리**: 프로젝트 루트 `.env`의 `GEMINI_API_KEY`만 사용. 키 값은 로그·콘솔·보고서·예외
  메시지 어디에도 노출하지 않음.
- **결정론**: temperature 0. 2.5 Flash류의 thinking은 비활성(속도·결정성 우선).
- **안정성**: API 타임아웃 · 429/5xx 지수 백오프 재시도 · **네트워크/재시도 소진 시 로컬 12b 자동 폴백**
  (전 구간 로컬로 돌아가는 길을 항상 유지).
- **무료 티어 한도 준수**: 실측 20 RPM → 프로세스 전역 3.2s 최소 호출간격 스로틀(429 예방).
- **비용 실측**: 호출마다 입력/출력 토큰(usageMetadata) 기록 → 질문당 비용 계측.

### 2.4 모델 선택 (계획 대비 불가피한 변경)
계획은 `gemini-2.5-flash`를 지정했으나, 현재 API 키(신규 프로젝트)에는 **404 "no longer available to
new users"** 로 호출 자체가 불가(모든 이름 variant · v1/v1beta 확인). 2.0-flash 계열은 429("check your
plan")로 무료 불가. **사용자 결정으로 최저가·최고 무료한도 tier인 `gemini-3.1-flash-lite` 채택.**
결과적으로 더 싸고 더 빠르게 동작.

## 3. 성능 비교 (동일 평가셋 실측)

### 3.1 표준 36문항 회귀셋

| 비교 축 | 로컬 RAG3 (A) | **Gemini Flash-lite (C)** |
|---|---|---|
| avg_kw_hit (관련 31) | 0.528 | **0.719** (+0.19) |
| 내용기반 답변율 | 0.645 | **0.903** |
| page_hit@3 / doc_hit@3 | 0.774 / 1.0 | 0.774 / 1.0 *(검색 동일)* |
| **환각 / 무관거절** | **0 / 100%** | **0 / 100%** |
| 파싱천장 미답 5문항 개방 | 0/5 | **4/5** |
| 평균 / 중앙값 / 최대 지연 | 63.4 / 48.3 / 148.6s | **6.2 / 6.4 / 13.0s** |
| 전 문항 20초 이내 | ✗ | **✓ (36/36)** |
| 질문당 비용 | ₩0 | ~$0.0004 |

- **속도**: 로컬 12b는 "호출당 10~15초 × 다중 호출"이 시간의 95%를 차지하고, 콜드 상태에서 공백 토큰이
  폭주하는 length-retry 꼬리(최대 148.6s)가 있었다. Gemini는 이 현상이 **원천 소멸**해 전 문항 13초 이내.
- **정확도**: avg_kw_hit 0.528 → 0.719. 특히 기존에 파싱 한계로 답을 못 하던 5문항 중 **4개가 근거 기반
  정답으로 전환**(환각 아님, 미지원 수치 0건).
- **절대지표 불변**: 환각 0, 무관질문 거절 100%는 어떤 경우에도 후퇴하지 않음(무관 거절은 로컬 검색
  점수 기준으로 결정되어 모델과 무관).

### 3.2 (참고) 다문서 종합·추론 12문항 — 신규 평가셋

| 축 | 로컬 12b | Gemini Flash-lite | Gemini + 분해검색·문장인용 |
|---|---|---|---|
| avg_kw_hit | 0.882 | 0.921 | 0.916 |
| 문서 간 비교 2문항 | 0.625 | 0.625 | **0.917** |
| 평균 지연 | 68.0s | **6.9s** | 10.3s |

- 종합형에서도 정확도는 근접하나 로컬 12b는 **10배 느리고 변동이 큼**(한 문항 177초 데드라인 도달).
- "여러 문서를 걸쳐 종합"하는 질문은 분해검색 옵션이 추가 이득(문서간 0.625→0.917). 이 평가셋의 정답은
  **자체검증 상태**로, 사용자 검수 확정 시 재측정 예정(경향은 불변 예상).

## 4. 불변 보장 (기존 자산 보호)

- **기존 rag3 코드 무수정**: 모든 구현은 `rag3x/`에. 실험 플래그 OFF = rag3와 byte 수준 동작 등가(실측 증명).
- **디스크 포맷 불변**: `vectors.npz`·`docs.json`·`page_store.json` 그대로 재사용(읽기 전용).
- **API 계약 불변**: `Rag3xEngine.ask()` 반환은 기존 `Rag3Engine`과 동일 키 + Gemini 계측 키만 추가.
- **폴백 경로**: 외부 API 장애 시 자동으로 로컬 12b로 답변 → 서비스 연속성 유지.

## 5. 사용법 — `ask_cli_gemini.py` (확인용 질의 CLI)

기존 `ask_cli.py`의 Gemini 버전. 검색·리랭커·임베딩은 로컬, 답변·검증만 Flash-lite.

```bat
:: intern_chatbot 환경 + 프로젝트 루트 .env에 GEMINI_API_KEY 필요
cd test_3
python ask_cli_gemini.py                       # 대화형
python ask_cli_gemini.py -q "제주교육청 스쿨넷 5단계 DR 100M 요금은?"
python ask_cli_gemini.py --file 질문.txt        # 한글이 콘솔에서 깨질 때
```

출력 예시(실측): 답변 + 근거 페이지 + `[Gemini] 호출 2회 · 토큰 in 9695/out 52 · API 1.9s · 비용≈$0.00099`
+ `[소요] 8.8s`. 로컬 `ask_cli.py`와 나란히 돌리면 속도 차이를 즉시 체감할 수 있다.

## 6. 한계·주의

1. **외부 의존**: 인터넷·API 필요(무료 20 RPM). 단 장애 시 로컬 12b 폴백으로 가용성 확보.
2. **비용 요율은 가정치**: 토큰 수는 정확히 실측되나 단가(lite tier in $0.10 / out $0.40 per 1M)는
   공개 요율로 검증 필요. 무료 티어면 사실상 ₩0.
3. **데이터 외부 전송**: 질문·근거 텍스트가 Google API로 전송됨(사용자 보안 검토 승인 완료 전제).
4. **종합형 평가셋 검수 대기**: §3.2 수치는 자체검증 정답 기준. `PHASE6_SYNTH_EVAL_DRAFT.md` 검수 후 확정.

## 7. 결론

로컬 RAG3의 **검색·검증 신뢰성(환각 0 · 무관거절 100% · 검색 0.774)을 그대로 유지**하면서, 생성·검증만
Gemini Flash-lite로 치환하는 것만으로 **응답속도 ~10배(63.4s→6.2s, 전 문항 20초 이내) · 정확도 향상
(0.528→0.719) · 파싱천장 4/5 개방**을 동시에 얻었다. 질문당 비용은 약 0.5원 수준. 기존 코드·인덱스·API
계약은 전부 불변이며, 외부 API 장애 시 로컬로 폴백된다. **"외부 API RAG가 로컬 대비 이만큼 빠르다"는
대조군으로서, 로컬판과 동일 파이프라인·동일 검색 위에서 LLM만 바꾼 순수 비교를 제공한다.**
